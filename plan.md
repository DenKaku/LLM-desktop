# 做一个和本地部署的LLM模型聊天的桌面应用

## 系统环境

 - linux ubuntu24

## LLM环境

 - 本地通过ollama部署的LLM

## 项目构建需求

 - 使用electron
 - 使用当前文件夹自行创建electron项目
 - 在当前文件夹内进行代码编写
 - api文档参照 https://docs.ollama.com/api/introduction
 - 切换模型时，同时清除上下文。

## 画面需求

- 参照微信聊天界面
- 需要可以选择通过ollama部署好的LLM模型的下拉菜单
- 可以选择关闭或选择思考模式程度的下拉菜单，如若用户选择的模型没有思考模式，默认选择为关闭
- 在关闭思考模式的情况下，聊天应答loading时不要显示“思考中”。
- 在开启思考模式时显示思考中。
- 思考中的要素，在显示思考内容的element的左上角，并以小字显示。
- 在思考模式时，需显示思考内容

## 代码优化

- 所有关于type和interface的定义，新建文件夹types，并且在types文件夹里新建相应文件，把type和interface定义迁移到新的文件。采用import方式进行调用。

## 画面需求增加

- 增加外观light和dark模式选择。
- 滚动条样式参照mac系统
- 增加可上传本地文件的“+”按钮，并可以实现本地添加文件功能
